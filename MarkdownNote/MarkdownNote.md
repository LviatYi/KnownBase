## 1. 标题

Atx风格标题  

用```###```表示标题  
用```#```的数量表示标题的次序  

> # 1 我是一级标题
> 
> ## 2 我是二级标题
> 
> ### 3 我是三级标题
> 
> #### 4 很明显，我是四级标题
> 
> ##### 5 那么我就是五级标题
> 
> ###### 6 我是快看不见的六级标题

``` Markdown
# 1 我是一级标题
###### 6 我是六级标题
```

Setext风格标题（不推荐）  

```===```（一级标题）  
与  
```---```（二级标题）  
（任意长度都可）  

> 这是一级标题
> =============
> 
> 这是二级标题
> -------------

```Markdown
这是一级标题
=============

这是二级标题
-------------
```

## 2. 段落

在 Markdown 中，段落用两个以上空格加一个回车表示  

也可以用一个空行来表示一个段落

> 我是段落一  
> 我是段落二  

> 我是段落一
> 
> 我是段落二

``` Markdown
我是段落一  
我是段落二  

我是段落一

我是段落二

```

不同的 Markdown 处理器可能出现不同的效果  
在本文档中，统一使用两个空格加一个回车表示段落  


## 3. 块引用
Markdown 使用email样式的「 > 」字符作为块引用  

> 我就是块鸭  
> 块块块块块  
> 块块块块块  
> 在块中 每行末尾都需要强制换行  

块引用中支持嵌套，只需要多加几个「 > 」即可

>> 这是块中块  
>>> 这是块中块中块  
>>>> 这是块中块中块中块  
``` Markdown
>> 这是块中块  
>>> 这是块中块中块  
>>>> 这是块中块中块中块  
```

块引用中可以包含 Markdown 元素  

> 块中的 Markdown 元素:  
> </br>
> ## 2 我是二级标题  
> </br>
>   
> ``` C
> int main()
> {
>    cout>>「 我是后面会讲到的代码块 」>>endl;
> }
> ```

``` Markdown
> 块中的 Markdown 元素:  
> ## 2 我是二级标题  
>   
> ``` C
> int main()
> {
>    cout>>「 我是后面会讲到的代码块 」>>endl;
> }
> ```
```

## 4. 列表

Markdown 支持有序列表和无序列表  

无序列表可以用 ```+``` ```-``` ```*```  

> + Red  
> + Green  
> * Red  
> * Green  
> - Red  
> - Green  

```Markdown
+ Red  
+ Green  
* Red  
* Green  
- Red  
- Green  
```

有序列表可以用  ```数字+句号```  

> 1. 你好  
> 2. 我好  
> 3. 大家好  

```Markdown
1. 你好  
2. 我好  
3. 大家好  
```

输出的列表序号仅与 Markdown 代码中的第一个序号相关，而与其余的无关。  

如果你需要一个从3开始的有序列表：  

> 3. 大家好  
> 4. 家好  
> 5. 好  

``` Markdown
3. 大家好  
4. 家好  
5. 好  
```

甚至，你可以这样写：  

``` Markdown
3. 大家好  
6. 我是乱七八糟的 Markdown 代码  
666. 但是我却能输出正常的序号  
```

> 3. 大家好  
> 6. 我是乱七八糟的 Markdown 代码  
> 666. 但是我却能输出正常的序号  

（当然，没有逻辑的写法也许会遭到来自其他开发者的毒打）

如果块注释被包含在列表项中，应该使用缩进表达这种逻辑关系  
代码块同理  

> 1.  我是列表项1  
> 2.  我是列表项2  
>     > 我是列表项2包含的块  
>     
>     我是列表项2包含的块包含的代码段（好绕）  
>     ``` C
>     int main()
>     {
>         cout<<「 Hello world again 」<<endl;
>         return 0
>     }
>     ```
> 
> 3. 我是列表项3  

``` Markdown
1.  我是列表项1  
2.  我是列表项2  
    > 我是列表项2包含的块  
    
    我是列表项2包含的块包含的代码段（好绕）  
    ``` C
    int main()
    {
        cout<<「 Hello world again 」<<endl;
        return 0
    }
    ``` （请忽视本括号内的内容）

3. 我是列表项3  
```
如果在编辑中出现了格式问题，没有什么是比来一次缩进更方便的解决办法了，如果有，那就是「再来一次」  

## 5. 代码块

代码块可以潇洒地输出代码而不用担心与Markdown代码的冲突问题  

（不推荐）直接使用四个空格或一个 `Tab` 即可表示不指定语言代码块  

>     int main()
>     {
>         cout << 「 Hello World! 」 <<endl;
>         return 0;
>     }
> 聪明的人已经发现这是一段C++代码

```Markdown
    int main()
    {
        cout << 「 Hello World! 」 <<endl;
        return 0;
    }
```

（推荐）也可以用「 ``` 」表示不指定或指定语言的代码块  
如果指定了语言，在不同的渲染器中，甚至可以渲染出炫酷的染色  

``` C
int main()
{
    cout << 「 你已经看到我好几次啦 」 << endl;
    return 0;
}
```

```` Markdown
``` C
int main()
{
    cout << 「 Hello World! 」 <<endl;
    return 0;
}
```
````

如果不指定语言，那么块前与块后的两个「```」标记可以出现在同一行  

> 这样就可以像这样展示一句 ```cout<<「行内代码」<<endl;```  

``` Markdown
行内代码这样写： ```cout<<「行内代码」<<endl;```  
```

如果需要添加行数信息，请添加{.line-numbers} class  

```C {.line-numbers}
int main()
{
    cout << 「 你已经看到我好几次啦 」 << endl;
    return 0;
}
```

````Markdown
```C {.line-numbers}
int main()
{
    cout << 「 你已经看到我好几次啦 」 << endl;
    return 0;
}
```
````

可以通过添加 highlight 属性指定行地使代码高亮

```C {.line-numbers,highlight=[2-3,5]}
    cout << 「 高亮 就是变亮吗 」 << endl;
    cout << 「 哇 真的变亮了 」 << endl;
    cout << 「 耶吼 我被高亮啦!! 」 << endl;
    cout << 「 不要高亮我（弱） 」 << endl;
    cout << 「 请问我被高亮了吗 」 << endl;
```

````Markdown
```C {highlight=[2-3,5]}
    cout << 「 高亮 就是变亮吗 」 << endl;
    cout << 「 哇 真的变亮了 」 << endl;
    cout << 「 耶吼 我被高亮啦!! 」 << endl;
    cout << 「 不要高亮我（弱） 」 << endl;
    cout << 「 请问我被高亮了吗 」 << endl;
```
````

你可以添加「 ` 」的个数来自定义代码段的头尾标记以避免一些冲突，这样就可以在代码段中放置「 ``` 」  

## 6. 字体修饰

Markdown 可以使用以下的标记字体  

斜体  
> *70°优雅的斜角表现了我的高贵气息*  
> _通常用于补充解释，或引用名人名言/作品名等场景_  

``` Markdown
*文本*  https://t.bilibili.com/https://t.bilibili.com/
_文本_  
```

粗体  
> **又粗又*的我可以拖慢你的阅读速度**  
> __从而起到强调文本的作用__  

``` Markdown
**文本**  
__文本__  
```

粗斜体  

> ***斜体+粗体，如果不是这行字特别爱表现自己，那么就一定是它非常的重要***  
> ___比如：我还没有女朋友（暗示）___

``` Markdown
***文本***
___文本___
```

为了让文档更具实用意义，应该合理（克制）地使用粗体与斜体  

删除线  
> ~~请给我分配一个女朋友（不准明示）~~

```Markdown
~~文本~~
```

下划线  
> <u>写上面那句话的人真是有够不要脸的</u>  

```Markdown
<u>文本</u>
```

脚注  
脚注的格式为[\^x] 其中 x 可以是数字、字母或汉字、单词或词组  
脚注必须与其引用所对应。  

> 本文作者：@我惘故我在[^我惘故我在] 是一个非常帅的[^2]（未来）全栈游戏开发者[^3]  

[^我惘故我在]: 一个自命不凡且不自知的人  
[^2]: 不自知  
[^3]: 自命不凡  

``` Markdown
......[^AAA]......[^2]......[^3]......  

[^AAA]: text1  
[^2]: text2  
[^3]: text3  

```

想知道脚注在哪？  

脚注的说明部分将被自动移动至文档末尾  
你可以通过点击引用标号或下拉至文档末尾找到它们  

## 7. 链接

你可以在 Markdown 文件中藏匿指向外部的链接  

先介绍两种添加方式：  

> [仇人生病了怎么办？](https://www.baidu.com)  
>  
> 今天你打卡了吗？没有的话请去 <https://www.github.com>  

``` Markdown
[显示文本](链接)  

其他文本<链接>  

```

另外还有变量式添加方式  
这样可以先声明变量，在之后添加链接：  
> [点我进入Trump的办公室][TWITTER]  
>  
> [TWITTER]:https://www.twitter.com  

```Markdown
[显示文本][变量名]  

[变量名]:链接  

```

## 9. 表格

Markdown 的表格制作非常形象。你可以用 ``` | ```直接画出表格  

> |  表头  |  表头  |
> |  ----  | ----- |
> | 单元格 | 单元格 |
> | 单元格 | 单元格 |

```Markdown
|  表头  |  表头  |
|  ----  | ----- |
| 单元格 | 单元格 |
| 单元格 | 单元格 |

或懒鬼写法

  表头  |  表头  
  ----  | ----- 
 单元格 | 单元格 
 单元格 | 单元格 
```

可以在隔断层加入标记 以控制所在列的对齐  

> |  默 认 左 对 齐  |  左 对 齐  |  居 中 对 齐  |  右 对 齐  |
> |  -------------  | :--------- | :----------: | --------: |
> |       左边       |    左边    |     中间     |    右边    |
> |       左边       |    左边    |     中间     |    右边    |

```Markdown
|  默 认 左 对 齐  |  左 对 齐  |  居 中 对 齐 |  右 对 齐  |
|  -------------   | :--------- | :----------: | ---------: |
|       左边       |    左边    |     中间     |    右边    |
|       左边       |    左边    |     中间     |    右边    |
```

代码中的 空格 和 ```|``` 其实只需要一个，不过作者有严重的强迫症
